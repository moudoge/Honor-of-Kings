# Honor of Kings - 围城补给行动

> 本作品为 **2026 中国服务外包大赛 D 类赛题：【D05】电竞AI应用创新与实践【腾讯开悟】** 的参赛作品。

## 1. 项目简介

本项目是一个基于 Phaser 3 + TypeScript 的 2D 俯视角战术补给行动 Demo。玩家扮演百里守约，在长城外圈高压战场中完成“搜集关键补给 -> 回城提交 -> 行动结算”的完整任务闭环。

项目同时集成“角色对话 AI（腾讯混元兼容接口）”，可与花木兰、铠、百里玄策进行战术交流，回复内容会结合当前战场状态、补给进度和历史对话。

## 2. 技术栈与运行环境

- 引擎：Phaser 3.90.0
- 语言：TypeScript 5.9
- 构建工具：Vite 8
- AI 接入方式：Vite 中间件代理 `/api/npc-dialogue` -> 混元兼容 `chat/completions` 流式输出

## 3. 场景流程（完整玩法链路）

项目包含以下场景（`src/main.ts`）：

1. `MainMenuScene`（主菜单）
2. `TrainingRoomScene`（预留训练场景）
3. `CharacterSelectScene`（角色选择）
4. `BriefingScene`（出城前简报）
5. `MissionScene`（局内行动）

标准流程：

1. 首页 -> 进入任务准备
2. 选择行动模式 + AI 开关
3. 进入角色选择（当前仅百里守约可用）
4. 查看简报 -> 开始行动
5. 局内搜集关键资源并回主城提交
6. 胜利进入结算界面 / 失败进入失败弹窗

## 4. 主菜单与任务准备特性

### 4.1 主菜单

- 动态背景、前情弹窗、角色立绘展示
- `查看围城前情` 面板
- BGM 音量滑条（支持拖动，持久化到 `localStorage`）
- 键盘快捷：
  - `Enter` 打开任务准备
  - `Esc` 关闭当前弹层

### 4.2 任务准备

- 两种行动规格：
  - 围城应急行动（`standard`）
  - 围城极限行动（`pressure`）
- 战场 AI 开关：
  - 开启后可在局内与 NPC 进行智能对话
  - 关闭后仅保留静态提示

## 5. 角色选择系统特性

### 5.1 角色卡机制

角色选择界面提供 4 张角色卡：

- 铠（未解锁）
- 百里守约（可选择）
- 百里玄策（未解锁）
- 花木兰（未解锁）

当前版本设计为：**仅百里守约可进入下一步**。

### 5.2 悬浮信息浮窗

鼠标悬浮角色立绘，会显示：

- 职业
- 角色特点
- 台词（引号样式）

浮窗会根据文本长度自动调整宽度与换行，避免文字溢出。

### 5.3 交互反馈

- 未解锁角色点击提示：`<角色名> 暂未解锁，请选择其他角色。`
- 未选择可用角色时，下一步按钮不可进入。

## 6. 出城简报与剧情提示

- 简报根据模式和 AI 开关动态生成文本。
- 局内开场连续播报：
  - 花木兰军令
  - 百里玄策侦报
  - 战场通报（仅 AI 开启时额外显示）

## 7. 局内核心玩法特性

## 7.1 地图与战区

- 世界尺寸：`4200 x 2600`
- 长城内侧安全区 + 城门前线 + 外围补给区 + 高压敌区
- 地图内包含：
  - 后方营地（含帐篷）
  - 后营篝火（可交互合成）
  - 主城物资提交处（胜利判定关键）
  - 草丛潜伏区、补给点、障碍碰撞区

## 7.2 出生点

- 玩家出生在后营篝火左侧（由 `PLAYER_SPAWN_POSITION` 定义）

## 7.3 HUD 显示模式

- 局内默认 **隐藏详细界面**（`hudCollapsed = true`）
- `H` 可切换“显示详细信息/隐藏详细信息”
- 隐藏时保留关键提示；显示时展开完整状态板

## 7.4 基础操作

- `W/A/S/D`：移动
- `Shift`：冲刺
- 鼠标左键：射击
- `R`：换弹
- `Q`：静谧之眼（扫描显形）
- `E`：狂风之息（架枪狙击）
- `Space`：后撤反击
- `F`：交互（NPC / 补给 / 篝火 / 提交处）
- `Tab`：背包与补给面板
- `5`：使用恢复品
- `H`：切换界面显示状态

## 7.5 武器与战斗数值（当前版本）

主武器：狙击步枪

- 伤害：280（普通）
- 射击间隔：430ms
- 弹匣：5
- 预备弹药：24
- 换弹时间：1600ms
- 子弹速度：820
- 普射距离：420
- 狙击距离：1280

附加机制：

- 子弹为 0 时再次射击会提示：`弹药不足，请换弹`
- 换弹显示角色头顶进度条（不锁移动）

## 7.6 技能系统

### Q 静谧之眼

- 扫描半径：360
- 持续：9s
- 冷却：18s
- 功能：显形草丛内敌人，提升信息掌握能力

### E 狂风之息

- 架枪时长：4.5s
- 冷却：12s
- 再按 E 可提前取消且不消耗冷却
- 架枪射击为高威胁狙击弹道与特效

### Space 后撤反击

- 冷却：8s
- 短时后撤位移 + 自动反击射击
- 适用于被贴脸时快速拉扯

## 7.7 敌人系统

当前可见敌对逻辑包括：

- 杂兵（grunt）
- 精英（elite）
- 石像精英（猩红/蔚蓝）
- Boss 逻辑模板（代码已定义，当前默认流程未固定投放）

动态机制：

- 压力值系统（敌人数、血线、时间共同影响）
- 定时增援（不同模式节奏不同）
- 石像精英按阶段刷新，击败后掉落高价值补给
- 草丛潜伏、扫描显形、追击/分离/避障等行为

## 7.8 NPC 系统与行为

三名友军 NPC：

- 花木兰：统帅 / 防线与节奏管理
- 铠：前线支援 / 抗压护送
- 百里玄策：外线侦查 / 路线联动

NPC 支持：

- 巡逻路线
- 状态机切换（patrol/guard/warn/support/...）
- 战场态势触发的自动台词
- 避免抖动的朝向稳定策略（小位移阈值控制翻转）

## 8. 资源与任务系统

## 8.1 补给分类

- 食物类：军粮袋、应急口粮、肉干、赤甲肉、熟食
- 医疗类：守城医疗包、绷带卷、草药包
- 生存类：净水壶、火种包、工具包
- 军需类：狙击专用弹、箭矢束、守城器械零件

## 8.2 关键资源判定

关键资源（计入目标）包括：

- 军粮类关键物资
- 守城医疗包
- 守城器械零件

## 8.3 胜利条件（当前实现）

> 当前逻辑不是“拿够即结束”，而是“拿够后还要提交”。

流程为：

1. 收集到目标份数关键补给（模式决定份数）
2. 回主城 `物资提交处`
3. 按 `F` 提交成功后，才进入结算界面

## 8.4 篝火合成系统

- 交互对象：后营篝火
- 配方：`3 片赤甲肉 -> 1 份熟食`
- 交互流程：
  1. 第一次 `F` 打开确认提示
  2. 再次 `F` 执行合成
- 若材料不足，提示：`赤甲肉不足，请在野外搜索。`

## 8.5 恢复道具系统

恢复优先级：

1. 熟食（+32）
2. 草药包（+15）
3. 绷带（+5）

规则：

- 按 `5` 使用
- 使用时角色停止移动 2 秒
- 角色头顶显示进度条
- 结束后立即结算回血并弹出提示

## 9. 交互系统

`F` 统一交互支持：

- NPC 对话
- 补给箱搜索
- 篝火合成
- 主城提交

并提供：

- 交互环高亮
- 世界坐标提示文本
- 交互优先级自动选最近有效目标

## 10. 角色对话 AI 特性（腾讯混元兼容）

## 10.1 对话入口

- 在 NPC 附近按 `F` 打开战术频道
- 输入文本后走流式回复
- 支持 Enter 发送 / Esc 关闭

## 10.2 上下文注入

每次请求会携带：

- NPC 人设
- 当前战术提示
- 当前压力值
- 当前补给进度
- 最近对话历史
- 持久化短记忆摘要

## 10.3 提示词约束

服务端会在系统提示中约束：

- 始终保持角色身份
- 先回答问题，再给建议
- 2~5 句中文短句
- 禁止跳出角色谈模型/系统

## 10.4 可用性保障

- 支持流式解析 SSE 数据块
- AI 关闭时给出系统提示，不发起请求
- 请求失败时在对话框给出错误提示，不中断游戏主流程

## 11. 胜负与结算表现

## 11.1 失败流程

- 角色 HP 归零后不直接返回首页
- 弹出失败弹窗，提供：
  - 再试一次
  - 返回首页

## 11.2 胜利流程

- 胜利进入专用结算界面（背景图：`mission_result_bg2.webp`）
- 结算页不灰化背景
- 右下角提供“回到首页”按钮

## 13. 本地运行

### 13.1 安装依赖

```bash
npm install
```

### 13.2 配置环境变量（可选开启 AI）

复制 `.env.example` 为 `.env`：

```env
HUNYUAN_API_KEY=your_hunyuan_api_key_here
HUNYUAN_BASE_URL=https://api.hunyuan.cloud.tencent.com/v1
HUNYUAN_MODEL=hunyuan-turbos-latest
```

### 13.3 启动

```bash
npm run dev:local
```

局域网联调：

```bash
npm run dev
```

### 13.4 构建与预览

```bash
npm run check
npm run build
npm run preview
```
